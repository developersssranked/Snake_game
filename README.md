# Змейка — курсовая работа (Python + Pygame)

Классическая игра «Змейка» с разными типами яблок, локальной базой
результатов, таблицей рейтинга и редактором уровней. Проект написан так,
чтобы наглядно демонстрировать принципы объектно-ориентированного
программирования.

## Запуск

```bash
pip install -r requirements.txt
python main.py
```

## Меню

Одна кнопка **«Играть»** запускает игру на последнем созданном уровне.
Уровень всегда один: новое сохранение в редакторе **заменяет** старое.
Если уровней ещё нет — игра идёт на пустой карте («Классика»).

## Управление

**В игре:** стрелки или WASD — движение, ESC — выход в меню.

**В редакторе уровней:**
- ЛКМ — поставить/убрать стену
- ПКМ — задать стартовую позицию змейки
- `C` — очистить все стены
- `S` — сохранить уровень (ввести название)
- ESC — выход в меню

## Типы яблок

| Яблоко       | Иконка   | Эффект                          | Очки |
|--------------|----------|---------------------------------|------|
| Обычное      | 🍎        | +1 сегмент                      | +10  |
| Золотое      | ⭐        | +2 сегмента (редкое)            | +30  |
| Отравленное  | 💀        | укорачивает змейку              | −15  |

Иконки — спрайты из папки `assets/` (Twemoji, CC-BY 4.0). Если файлов
нет, игра рисует яблоки кружками (запасной вариант в `assets.py`).

## Принципы ООП в коде

- **Абстракция** — `game/base.py`: абстрактный базовый класс `GameObject`
  с абстрактным методом `update()`. Создать его напрямую нельзя.
- **Наследование** — `Snake`, `Apple`, `Wall` наследуют `GameObject`;
  `NormalApple`, `GoldenApple`, `PoisonApple` наследуют `Apple`.
- **Полиморфизм** — у каждого яблока свой `apply_effect()`. Игровой цикл
  (`game/game.py`) вызывает `apple.apply_effect(...)`, не зная конкретного
  типа. Метод `draw()` тоже переопределён по-разному (змейка, круг-яблоко,
  стена).
- **Инкапсуляция** — приватные поля (`_position`, `_body`, `_score`) и
  доступ к ним через `@property`; вся работа с БД спрятана внутри класса
  `ScoreDatabase`.

## Структура проекта

```
main.py              — точка входа
game/
  settings.py        — константы и цвета
  base.py            — GameObject (абстрактный базовый класс)
  snake.py           — Snake
  apples.py          — Apple → NormalApple / GoldenApple / PoisonApple
  wall.py            — Wall
  level.py           — модель уровня + сохранение/загрузка (JSON)
  database.py        — SQLite: результаты и рейтинг
  assets.py          — загрузка и кэширование иконок (+ запасная отрисовка)
  ui.py              — кнопки, текст, градиентный фон
  game.py            — игровой цикл
  editor.py          — редактор уровней
  menu.py            — меню и таблица рейтинга
assets/              — иконки яблок (apple_red.png, star.png, skull.png)
levels/              — сохранённый уровень level.json (создаётся автоматически)
scores.db            — база результатов (создаётся автоматически)
```

## База данных

Используется SQLite (встроена в Python). Таблица `scores`:
`id`, `player`, `score`, `level`, `length`, `played_at`.
Результат записывается после каждой партии; рейтинг выводится в меню
(топ-10 по убыванию очков).
