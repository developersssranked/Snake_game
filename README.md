# Змейка — курсовая работа (Python + Pygame)

Игра «Змейка» с партией по таймеру, набором поведенчески разных бонусов,
разрушаемыми стенами, локальной базой результатов, таблицей рейтинга и
редактором уровней. Проект написан так, чтобы наглядно демонстрировать
принципы объектно-ориентированного программирования.

Цель — набрать максимум очков, пока не кончилось время. Бонусы «Часы»
добавляют время; разные бонусы по-разному меняют состояние игры.

## Запуск

```bash
pip install -r requirements.txt
python main.py
```

## Меню

Одна кнопка **«Играть»** запускает игру на последнем созданном уровне.
Уровень всегда один: новое сохранение в редакторе **заменяет** старое.
Если уровней ещё нет — игра идёт на пустой карте («Классика»).

## Управление

**В игре:** стрелки или WASD — движение, ESC — выход в меню.

**В редакторе уровней:**
- ЛКМ — поставить/убрать стену
- ПКМ — задать стартовую позицию змейки
- `C` — очистить все стены
- `S` — сохранить уровень (заменяет прежний)
- ESC — выход в меню

## Бонусы (экран «Бонусы (справка)» в меню)

| Бонус             | Иконка | Поведение                                                   | Очки |
|-------------------|--------|-------------------------------------------------------------|------|
| Обычное яблоко    | 🍎     | +1 сегмент                                                  | +10  |
| Золотая звезда    | ⭐     | +2 сегмента (редкая)                                        | +30  |
| Отравленное яблоко| 💀     | травит: змейка роняет какашки и теряет очки                 | −15  |
| Какашка           | 💩     | след яда; работает как стена — врезался, конец игры         | —    |
| Антидот           | 💊     | снимает яд; яд и какашки временно безвредны                 | +10  |
| Бомба             | 💣     | взрывает все стены в радиусе вокруг себя                    | +20  |
| Броня             | 🛡     | змейка «стальная» — какое-то время ломает стены собой       | +15  |
| Ускорение         | ⏩     | временно быстрее                                            | +5   |
| Замедление        | ⏪     | временно медленнее                                          | +5   |
| Часы              | ⏰     | добавляют время на таймере                                  | +5   |

Опасные бонусы (яд) всегда появляются вместе с обычным яблоком —
есть безопасный выбор. Активные эффекты и их остаток времени
показаны иконками на верхней панели; броня красит змейку «сталью», яд —
фиолетовым. Иконки — спрайты из `assets/` (Twemoji, CC-BY 4.0); если
файла нет, объект рисуется фигурой (запасной вариант).

## Принципы ООП в коде

- **Абстракция** — `game/base.py`: абстрактный базовый класс `GameObject`
  с абстрактным методом `update()`. Создать его напрямую нельзя.
- **Наследование** — `Snake`, `Bonus`, `Wall`, `Poop` наследуют
  `GameObject`; все бонусы (`NormalApple`, `Bomb`, `Armor`, `Antidote`,
  `Clock`, …) наследуют `Bonus`.
- **Полиморфизм** — ключевой момент: у каждого бонуса свой `apply_effect()`,
  и поведение **принципиально разное** — рост, взрыв стен, броня, яд,
  изменение скорости, добавление времени, мгновенный проигрыш. Игровой цикл
  (`game/game.py`) вызывает `bonus.apply_effect(snake, game)`, не зная типа;
  результат определяется классом объекта. `draw()` тоже переопределён
  по-разному (лента-змейка, бонус с иконкой, стена, какашка).
- **Инкапсуляция** — приватные поля (`_position`, `_body`, `_status`,
  `_points`) и доступ через `@property`/методы; вся работа с БД и с
  системой эффектов скрыта внутри классов `ScoreDatabase` и `Snake`.

## Структура проекта

```
main.py              — точка входа
game/
  settings.py        — константы, цвета, длительности эффектов
  base.py            — GameObject (абстрактный базовый класс)
  snake.py           — Snake + система временных эффектов
  bonuses.py         — Bonus → яблоки, бомба, броня, антидот, часы… + Poop
  wall.py            — Wall (разрушаемые стены)
  level.py           — модель уровня + сохранение/загрузка (JSON)
  database.py        — SQLite: результаты и рейтинг
  assets.py          — загрузка и кэширование иконок (+ запасная отрисовка)
  ui.py              — кнопки, текст, градиентный фон
  game.py            — игровой цикл, таймер, столкновения, эффекты
  editor.py          — редактор уровней
  menu.py            — меню, справка по бонусам, таблица рейтинга
assets/              — иконки бонусов (apple_red, star, skull, bomb, shield, …)
levels/              — сохранённый уровень level.json (создаётся автоматически)
scores.db            — база результатов (создаётся автоматически)
```

## База данных

Используется SQLite (встроена в Python). Таблица `scores`:
`id`, `player`, `score`, `level`, `length`, `played_at`.
Результат записывается после каждой партии; рейтинг выводится в меню
(топ-10 по убыванию очков).
